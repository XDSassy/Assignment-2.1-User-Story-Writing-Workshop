This repository contains user stories for a **Conference Room Booking System**, organized into **epics** and prioritized for Agile development. Each story defines:

- **Who** the user is  
- **What** functionality they need  
- **Why** it is important  

Stories include acceptance criteria, priority, dependencies, and technical/design notes.

### Epics and User Stories

1. **Room Booking & Management**  
   Stories related to employees booking rooms, recurring meetings, filtering by capacity, cancelling bookings, and selecting rooms with specific equipment.  

2. **Administrative Oversight & Reporting**  
   Stories for admins to view dashboards, resolve booking conflicts, and generate usage reports.  

3. **Operational Support**  
   Stories for facilities managers scheduling maintenance and receptionists assisting visitor bookings.
